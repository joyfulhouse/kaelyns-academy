# Adventure 2.0 Phase B — Slice B3: The Adaptive Generation Loop

**Date:** 2026-07-07
**Status:** Design approved; awaiting user spec review
**Parent design:** `docs/superpowers/specs/2026-07-01-adventure-2-design.md` (§8 AI practice, §9 curriculum)
**Predecessor slices:** B1 Life Skills Math, B2 Science & Nature, C1 Baseline Placement — shipped, live.
**Trigger:** pilot feedback — "the curriculum isn't actually adaptive; finishing a lesson produces no new material."

## 1. Context — the verified diagnosis

Three stacked causes, confirmed against code and the prod DB:

1. **No generate step in the adaptive flow.** The recommender (`src/lib/tutor/recommend.ts`)
   only cycles the fixed authored set: it re-serves authored activities while a
   skill is emerging and returns `[]`/`null` when everything is solid — the
   "next thing" card simply disappears. The AI generator is reachable ONLY from
   a manual per-activity "More, made just for me" reward-screen button
   (`ActivityHost.tsx`), capped at 3/session, and its output is **ephemeral** —
   played once, never surfaced as new material.
2. **Half the program's kinds are authored-only.** 5 of 11 kinds in
   `kaelyn-adaptive` (`seq-order`, `sort-categories`, `math-clock`,
   `math-money`, `math-measure`) have no `KIND_BRIEF`, so even the manual
   button 502s for them (and the client doesn't pre-check `isGenerableKind`,
   so it dead-ends into "Let's do that another time").
3. **The pilot played in guest mode.** Prod has zero learners/enrollments/
   attempts (only the 2 e2e fixtures). Guest mode is localStorage-only and the
   generation path is account-only by design — she could never have reached it.

Also relevant: both tutor routes (`TUTOR_FAST`/`TUTOR_RICH` in
`src/lib/ai/models.ts`) alias the same `ha-assist` (Qwen) route, and the
Word Study kinds' `skillsAffected()` emit legacy `phonics.*`/`reading.decodable`
slugs instead of the authored `word.*`/`vocab.*` tags the recommender gates on
([[wordstudy-skillsaffected-mismatch]]) — so that strand can't register mastery
from play at all. Prod has **zero attempts**, so fixing it now needs no
evidence migration.

## 2. Goal

Close the loop: **finishing a lesson produces fresh, durable, playable
variations of that lesson's exercises**, generated by DeepSeek V4 through the
LiteLLM gateway, surfaced as visibly-new material on the map, feeding the
recommender — under the same §8 bounded/validated/parent-gated posture as all
child-facing AI. In the same slice: make all 11 of the program's kinds
generable (with deterministic answer-key validators for the risky ones), fix
the Word Study skill-slug mismatch, and get the pilot onto a real account.

## 3. Model routes (ds4)

`src/lib/ai/models.ts`:

- `TUTOR_RICH = "ds4"` (DeepSeek V4, local dgx route — verified live on the gateway)
- `TUTOR_FAST = "ds4-fast"` (verified live)

All tutor generation moves to DeepSeek V4 (`MODEL_FOR_BAND`: ready→FAST,
stretch→RICH). `ha-assist` stays configured on the gateway but unused by the
tutor. Per-language overrides (`MODEL_FOR_LANGUAGE`) are untouched. No other
gateway change; `LITELLM_URL`/`LITELLM_API_KEY` wiring is already live in prod.

## 4. Data model — durable generated material

New table **`generated_activity`** (one migration):

- `id` (uuid pk), `learnerId` (FK learner, cascade)
- `programSlug`, `unitKey`, `lessonId` (text — locate the shelf on the map)
- `kind` (text), `title` (text), `config` (jsonb — schema-validated at write),
  `skillTags` (jsonb string[])
- `genModel`, `genRoute`, `genAt` (provenance — same trio as `attempt`)
- `createdAt`
- Indexes: `(learnerId, lessonId)` (the shelf read), `learnerId` (COPPA reads).

Learner-scoped private material: never touches the shared authored curriculum
or `program_version` (no version-pinning/marketplace interference).
**COPPA (§3.7):** added to `buildLearnerExport` (new `generatedActivities`
array) and covered by the learner delete cascade in the same PR. Health canary
gains the table's required columns.

## 5. The loop — trigger, shelf, play

### 5.1 Eager batch on lesson completion
When the client records the attempt that completes a lesson's last authored
activity, it calls a new server action **`ensureLessonPractice(learnerId,
programSlug, lessonId)`**. Server-side, in order (all inside the action):

1. Tenancy (account owns learner) + ACTIVE enrollment + both `aiPractice`
   gates (same checks as `/api/practice`).
2. Lesson-completion witness: every authored activity in that lesson (resolved
   from the learner's pinned tree) has a non-generated attempt.
3. Idempotency: if the lesson already has generated rows, return them
   (no duplicate batch).
4. Generate a batch of **4** items for the lesson's skills/kinds via the
   existing `generatePracticeItems` (band from enrollment config), each item
   schema-validated + kind-validated (§7), then persisted to
   `generated_activity` in one transaction.

The client shows the shelf in a "cooking…" state while the action runs
(~seconds on ds4) and swaps in the items when it resolves. Failures are calm
and silent-forgiving (the shelf just doesn't appear; `captureNonCritical`).

### 5.2 On-demand "More like this"
The shelf offers one bounded **"More like this"** affordance → the same action
with `more: true` → generates up to 4 more. Hard cap **8 generated items per
lesson**, enforced server-side (count before generate). The old per-activity
reward-screen "More" button remains, now gated on `isGenerableKind` so it never
renders for a kind that would 502.

### 5.3 Surfacing + play + credit
- **UnitView:** lessons with generated rows render a **"Fresh practice, made
  for you"** row after the authored activities (distinct look, sparkle icon,
  static classes) — visibly new material.
- **Recommender:** `nextBest` gains a generated-shelf source: when a strand has
  no fresh authored item, it recommends the learner's oldest uncompleted
  generated item (reason: "Fresh practice, made for you") before falling back —
  so the "next thing" card no longer goes blank.
- **Play route:** the activity page resolves authored first, then (account
  mode) a `generated_activity` row by id, learner-scoped. Completion records a
  real attempt with `generated: true` + provenance.
- **Stars:** a generated shelf item **earns stars once**, like authored. The
  star-mint gate gains a second server-verified membership witness: the
  activityId exists in `generated_activity` for THIS learner (the existing
  witness covers the authored tree). Best-star/once semantics keyed by the
  generated id, same as authored.
- **Guest mode:** unchanged — no generation, no shelf (account-only §8).

## 6. Generable-kind coverage — all five, with validators

`KIND_BRIEF` gains entries for `seq-order`, `sort-categories`, `math-clock`,
`math-money`, `math-measure` (making all 11 program kinds generable). Because
these are answer-key-sensitive, generation adds a per-kind deterministic
validator layer:

**New `validateGenerated?: (config) => string | null`** on the activity-type
contract (optional; null = valid, string = reason). Run server-side after zod
parse, before persist/return — for ALL generated items (the existing button
path too). Initial validators:

- `math-money`: the marked-correct choice equals the true coin sum
  (count mode: palette can reach `targetCents`).
- `math-clock`: `choices[answerIndex]` renders the stated `hour:minute`;
  choices unique.
- `math-measure`: `answerIndex` points at the true extreme for the stated
  attribute (compare mode) / the true `length` (units mode).
- `sort-categories`: every `item.binId` resolves (schema refine already) + no
  bin left empty.
- `seq-order`: structural only (3–6 unique cards); the brief constrains content
  to common-knowledge sequences (counting, size order, daily routine) — no
  niche facts. Residual risk accepted: a factually-wrong exotic sequence can't
  be caught deterministically, so the brief forbids exotic sequences.

An item failing validation is dropped; the batch retries once; a short batch
(even 0) is acceptable — a smaller shelf beats a wrong answer key.

## 7. Word Study `skillsAffected` fix

Pure code fix (prod has zero attempts — nothing to migrate):

- `phonics-wordbuild/logic.ts` `skillsAffected`: extend the `config.focus`
  substring mapping to emit `word.syllables.types`, `word.syllables.division`,
  `word.morphology.prefixes`, `word.morphology.roots` for the Word Study focus
  phrasings used in `kaelyn-adaptive`. Program-01 phonics focus strings keep
  mapping to `phonics.*` — behavior unchanged for that content.
- `sightword-game/logic.ts`: expand-only optional config field `skillTag`
  (zod `.optional()`); `skillsAffected` returns `[config.skillTag]` when
  present, else the legacy `["reading.decodable"]`. The Word Study content sets
  it (`vocab.shades-of-meaning`, `word.morphology.prefixes`).
- A focused test asserts every `kaelyn-adaptive` Word Study activity's
  `skillsAffected(config)` ⊆ its authored `skillTags` (the load-bearing
  placement/mastery invariant from C1, now extended to this strand).

This makes Word Study play credit the skills the recommender gates on — the
strand becomes genuinely adaptive, and the C1 Word Study baseline can return in
C2.

## 8. §8 posture (unchanged, extended)

- All generation server-side via LiteLLM; schema-validated + kind-validated;
  bounded (4/batch, 8/lesson, existing `MAX_ITEMS` guard); no open-ended chat.
- Parent gates (`settings.aiPractice`, enrollment `config.aiPractice`) enforced
  in `ensureLessonPractice` exactly as in `/api/practice`.
- Full provenance: `generated_activity` rows + `attempt.gen*` feed the existing
  parent "what the AI made" trail; the export gains `generatedActivities`.
- No child PII in any prompt (the existing interest-theming seam is unchanged).

## 9. Testing

- **Validators:** pure unit tests per kind (valid item passes; each corruption
  mode rejected).
- **Store/action:** `ensureLessonPractice` — tenancy, gates, completion
  witness, idempotency, the 8/lesson cap, one-transaction persist; star-mint
  witness for generated ids (earn once; foreign learner's id refused).
- **Recommender:** shelf items recommended when authored exhausted; blank-card
  case gone.
- **Word Study:** the `skillsAffected ⊆ skillTags` invariant test (§7).
- **e2e (LLM-free — the CI gate env has no LiteLLM):** guest sees no shelf and
  no More button; signed-in flow renders the More button on a generable-kind
  activity. No e2e may depend on live generation (assert affordances/gating
  only).
- Full gate before merge: `bun run lint && bun run typecheck && bun run test &&
  bun run build` + `bun run audit:dead-code`.

## 10. Deploy & rollout

- One DB migration (auto-runs via the migrate initContainer). No CI-gate
  change (no new seeded data; e2e is LLM-free). **Prod `seed-content.ts`
  re-run IS required**: the Word Study fix edits `kaelyn-adaptive.ts` (the
  sightword `skillTag` fields), and curriculum is DB-preferred.
- **Pilot rollout step (the guest-mode lesson):** create the real parent
  account + Kaelyn's learner + enrollment and sign her device in. The user
  creates the account (credentials stay theirs); we verify server-side that
  the learner + active enrollment exist and gates are on before calling it
  done. Canary includes: complete a lesson on a generable kind → the shelf
  appears with validated items.

## 11. Non-Goals (this slice)

- No change to the authored curriculum tree / marketplace / version pinning
  (generated material is learner-private).
- No open-ended chat; no interest-theming changes; no new activity kinds.
- No Word Study baseline unit (C2, after this slug fix ships).
- No auto-placement changes (C1 untouched); no mid/final checkpoints.
- No cloud ds4 routes (`ds4-cloud*`) — local first; swap is a one-line change.

## 12. Decision Log

| Decision | Choice | Alternatives |
|---|---|---|
| Scope | Everything in one slice (loop + ds4 + all 5 briefs + Word Study fix) | Core-loop-only (defer risky briefs + slug fix); trigger-only minimal |
| Persistence | `generated_activity` table + per-lesson "Fresh practice" shelf | Ephemeral virtual items; append to curriculum tree (rejected: pollutes shared content) |
| Trigger | Eager batch on lesson completion + bounded "More like this" | Eager-only; lazy-only |
| Model routing | `TUTOR_RICH="ds4"`, `TUTOR_FAST="ds4-fast"` | RICH-only ds4; cloud variants |
| Answer-key safety | Deterministic per-kind `validateGenerated` + drop-and-retry-once | Trust the model; LLM-as-judge (non-deterministic, slower) |
| Word Study fix | focus-mapping extension + expand-only `skillTag` config override | Change `skillsAffected` signature (ripples through the plugin contract) |
| Generated stars | Earn once, like authored (second membership witness) | No stars for generated (feels punitive; undermines "real material") |
